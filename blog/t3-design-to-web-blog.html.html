Responsive Web Design

16 December 2016

What a responsive site is, and why responsiveness is important?

Responsive Design is a technique of designing and developing a website to ensure it looks great on any screen and device. Previously designers and developers created separate pages specifically for mobile, tablet or desktop. This was a lot of work, as essentially 3 separate websites had to be built and maintained, as such responsive design provides not only an optimized user experience, but also saves developers and designers a lot of time as they only have one site to develop/maintain. Responsive design now allows designers and developers to make a single webpage that will display on any device. The content, styling and layout will all respond to the screen/device type ensuring the user is getting an optimal version of the site, no matter what device they are viewing it on.


What frameworks are, and their pros and cons?

Frameworks are a set of predefined website objects and layout stylings that developers can use when building a site. They allow developers to quickly develop the layout of their site as well as implement any desired features the framework supports. Frameworks provide a quick an easy way to develop or even prototype a website, however if you are just learning web design/development then they provide a barrier to student learning as they automate tasks that students would have to otherwise do themselves. Additionally, because frameworks are already predefined, they can also be somewhat restrictive or overwrite any other settings/stylings you may be using.


What mobile first design is, and why it's important?

Mobile First Design is the concept of designing and building a website starting with the mobile site. Previously designers have started with the desktop design, and then developed down to the mobile site. There are several issues with this method. Firstly, by doing it this way the mobile device often ends up having features cut or a user interface that isn't optimized for mobile. Given the continued growth in mobile traffic and ownership you want to ensure your mobile site provides a great user experience! With mobile first design, you start by designing the mobile site and make it as good as you possibly can to ensure the best possible experience, you then use that as your base and work up in size from there, to tablet and desktop. As the screen size gets bigger you can add more features to ensure the best experience on those devices as well - you now are providing your users with a fantastic experience on all devices. One other benefit of mobile first development, is that the mobile site will be faster to load as it's not loading all the tablet/desktop styling and images etc, whereas if you have a desktop first design, the mobile device is generally still loading these even if the image/feature isn't shown/being used, thus slowing the load time of the mobile site.


What a wireframe is and why do we use it?

Wire framing is important as it helps to give direction to the look and feel of the site, it helps designers and developers to understand how the site may look and operate before a line of code is typed or before Photoshop is opened! Wire framing can be used to quickly jot down new ideas around the sites design, user experience or features as well as can help justify what is important to the end user.

The aspects of your wireframes you found difficult to implement, and why?

I found it difficult to translate some ideas from mobile to desktop - I would have this idea of how something would look on mobile, but would then have to figure out how that would then look on desktop. The trickiest component was probably the about section that contained text and an image. The skeleton framework had a media query that kicked in too early and so didn't make this section look particularly great (long thin block of text), how I had to play around with the media queries to make sure that the text and image looked good at all screen sizes.
