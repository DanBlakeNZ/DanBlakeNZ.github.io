What 'scope' is, and how it works in JavaScript

Scope defines how/if javascript can access various types of variables, objects and functions. There is two types of scope, Local Scope and Global Scope. Say you have a variable which has Global Scope, this means that that variable can be accessed, and modified by a variety different functions and once the function has modified it in some way then the Global variable is still available and can be referenced at a later time or can be modified and/or accessed by other functions within your JavaScript code. By contrast Local Scope refers to something that can only be accessed temporarily, usually by one specific function. For example a variable with a Local Scope can usually be found located within a specific function. When that function is running, that variable might be used to store a temporary value that the function needs, however once the function has finished performing its task it will stop running and the local scope variable inside it and its value will be 'deleted' and will no longer be accessible. However if that same function runs again, then that local variable will once again be created and will be able to store any temporary values when said function is running.
