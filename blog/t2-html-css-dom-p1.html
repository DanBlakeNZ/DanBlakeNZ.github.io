Design and structural features I liked:

  • Some form of (relevant) full screen background image OR
  • Use of gradient colours
  • Navigation menu
  • Box layout -responsiveness - perhaps using flex boxes
  • Complementary Fonts
  • modern - styling ie more minimal than cluttered
  • Universal style - so is the same/familiar on each page
  • Styled Links (hover colour, visited colour etc) where relevant
  • Standardised headings across all pages
  • Buttons

How to describe HTML and CSS

The best, and most frequently used analogy is that HTML is frame of a house, it provides the structure and basic layout and content of the page, however on its own everything is pretty dull. CSS provides all the decoration - the colour, the styling, sizes, simple animations and transitions etc. Much like how paint, knickknacks and furnishings give go over the frame and give personality to a house, CSS gives personality to a webpage and makes it look good!


What is meant by boxifying design?

Given that webpages are frequently made up of may different boxes being able to boxify a design is a method of looking at a webpage design, and then figuring out how each section can fit into a 'box'. This can be done a number of ways, either on the computer using some form of editing program, or by printing off a design and using a pen, pencil or scissors.


What is the box model?

This is a model where each section of a webpage is made up one or more 'boxes' that contain certain content such as a logo, navigation bar, image, links, works, titles etc. The importance of box design is that it not only allows designers and developers to more easily move components around and develop a site, but it helps to ensure the web page will display correctly across a range of different screen sizes, and devices - ie PC, tablet and mobile.
